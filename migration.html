Migration Phases
Phase 1 — The Bridge 6–10 weeks
Move from raw Spring 4.3 to Spring Boot 3.5 on Java 17. The heaviest lift — javax→jakarta, XML config conversion, and JoinFaces integration.
  • Replace Spring modules with Boot starters + parent POM
  • javax.* → jakarta.* across all Java, XML, 80+ XHTML files
  • Integrate JSF & PrimeFaces via JoinFaces 5.x
  • Convert XML config → @Configuration (use @ImportResource as bridge)
  • Register web.xml filters & listeners as Spring beans
  • Update Oracle JDBC to ojdbc17 artifact
  • Adapt JUnit 5 tests to @SpringBootTest

Phase 2 — Major Upgrade 2–4 weeks
Upgrade to Spring Boot 4 and Framework 7. With jakarta migration done, focus on removed APIs, modularization, and Jackson 3.
  • Bump parent POM to Spring Boot 4.0.x
  • Handle Boot 4 modular starter renames
  • Fix removed Spring 5.x / 6.x deprecated APIs
  • Update @ConfigurationProperties (no public fields)
  • Validate Jackson 3 serialization behavior
  • Upgrade JoinFaces to 6.0.x for Boot 4 compat
  • Test virtual threads (enabled by default in Boot 4)
Phase 3 — Runtime Upgrade 1–2 weeks
Upgrade JDK from 17 to 25 LTS. Mostly a runtime swap with compatibility checks and optional language modernization.
  • Set java.version=25 in Maven POM
  • Run jdeps --jdk-internals for violations
  • Add --add-opens flags if needed for reflection
  • Verify Oracle JDBC driver on Java 25
  • Measure baseline performance (startup, memory, latency)
  • Optional: adopt records, pattern matching, sealed classes
